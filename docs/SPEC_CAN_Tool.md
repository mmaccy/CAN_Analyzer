## 1. 概要

### 1.1 目的

CAN / CAN FD バスのログファイル（Vector ASC 形式）を読み込み、DBC データベースファイルに基づいてフレーム・シグナルをデコードし、CANoe ライクな UI で解析・可視化を行うデスクトップツール。

### 1.2 ユースケース

| # | ユースケース | 説明 |
|---|---|---|
| UC-1 | ログ解析 | 記録済み ASC ファイルを読み込み、フレーム/シグナルの内容を確認する |
| UC-2 | フレーム抽出 | 指定フレームのみを抽出した ASC ファイルをエクスポートする |
| UC-3 | 時系列解析 | 特定シグナルの値の時間変化をグラフで可視化する |
| UC-4 | 統計分析 | フレーム周期・バスロード等の統計情報を確認する |
| UC-5 | リアルタイム受信 | Vector VN1610 経由で CAN FD バスをリアルタイムに受信・表示する |

### 1.3 対象ユーザ

S-CMS 開発チームのエンジニア（車載 CAN ネットワーク解析業務）

## 3. 機能要件

### 3.1 ASC ファイル読込

ASC ログファイル（Classic CAN / CAN FD 混在）をストリーミング方式で読み込む。

| 要件 | 説明 | 優先度 |
|------|------|--------|
| ASC-1 | Vector ASC 形式（.asc）の読込に対応する | 必須 |
| ASC-2 | Classic CAN フレームのパースに対応する | 必須 |
| ASC-3 | CAN FD フレーム（8〜64 バイト）のパースに対応する | 必須 |
| ASC-4 | 複数チャンネル（バス）のログを区別して扱う | 必須 |
| ASC-5 | 拡張 ID（29bit）フレームに対応する（ID 末尾 `x` マーカー） | 必須 |
| ASC-6 | GB 級ファイル（1GB+）をメモリ枯渇なく読み込む | 必須 |
| ASC-7 | ヘッダ情報（日時・base・version・UUID）を取得・表示する | 推奨 |
| ASC-8 | TriggerBlock の開始・終了を認識する | 推奨 |
| ASC-9 | ファイル読込の進捗をプログレスバーで表示する | 必須 |

### 3.2 CAN データベース読込（DBC / ARXML）

DBC または ARXML ファイルを読み込み、フレーム名・シグナル名・デコードルールを取得する。
ファイル拡張子（`.dbc` / `.arxml`）で形式を自動判別する。

| 要件 | 説明 | 優先度 |
|------|------|--------|
| DBC-1 | DBC ファイル（.dbc）の読込に対応する | 必須 |
| DBC-1a | ARXML ファイル（.arxml）の読込に対応する | 必須 |
| DBC-2 | 複数のデータベースファイルを同時に読み込める（DBC と ARXML の混在可） | 必須 |
| DBC-3 | フレーム ID ↔ フレーム名のマッピングを行う | 必須 |
| DBC-4 | シグナル定義（ビット位置・長さ・バイトオーダー・係数・オフセット）を取得する | 必須 |
| DBC-5 | シグナルの物理値変換（raw → physical）を行う | 必須 |
| DBC-6 | シグナルの単位・最小値・最大値を取得・表示する | 推奨 |
| DBC-7 | Value Table（列挙型シグナル）のデコードに対応する | 推奨 |
| DBC-8 | データベース未定義のフレームは ID と Raw データのみ表示する | 必須 |

### 3.3 トレースビュー

CANoe の Trace ウィンドウに類似した、フレーム一覧の時系列表示。

| 要件 | 説明 | 優先度 |
|------|------|--------|
| TRC-1 | タイムスタンプ・チャンネル・ID・フレーム名・DLC・データバイト列を表形式で表示する | 必須 |
| TRC-2 | DBC が読み込まれている場合、フレーム名を自動解決して表示する | 必須 |
| TRC-3 | フレーム ID またはフレーム名でフィルタリングできる | 必須 |
| TRC-4 | チャンネル（バス）でフィルタリングできる | 必須 |
| TRC-5 | テキスト検索（フレーム名・ID の部分一致）ができる | 必須 |
| TRC-6 | Rx / Tx 方向でフィルタリングできる | 推奨 |
| TRC-7 | フィルタ条件を組み合わせて適用できる（AND 条件） | 必須 |
| TRC-8 | 大量フレーム（100 万行+）をスクロールで快適に閲覧できる（仮想スクロール） | 必須 |
| TRC-9 | フレーム行を展開してシグナル値（物理値）を確認できる | 推奨 |
| TRC-10 | 特定フレームの行をハイライト色で区別できる | 推奨 |

### 3.4 シグナルツリー

DBC に基づくフレーム・シグナルの階層表示と選択 UI。

| 要件 | 説明 | 優先度 |
|------|------|--------|
| SIG-1 | DBC 内のフレームをツリー形式で一覧表示する | 必須 |
| SIG-2 | フレームを展開するとシグナル一覧を表示する | 必須 |
| SIG-3 | シグナルの詳細情報（ビット位置・長さ・factor・offset・単位・min/max）を表示する | 推奨 |
| SIG-4 | シグナルをチェックボックスで選択してグラフ表示対象にできる | 必須 |
| SIG-5 | フレーム名・シグナル名でツリーをフィルタリングできる | 必須 |
| SIG-6 | ログに存在するフレームのみハイライト表示する | 推奨 |
| SIG-7 | 選択中（チェック済み）シグナルは所属フレームを折りたたんでも常時表示し、選択解除できる | 必須 |

### 3.5 時系列グラフ

選択したシグナルの値を時間軸上にプロットする。

| 要件 | 説明 | 優先度 |
|------|------|--------|
| GRP-1 | 選択した複数シグナルを同一グラフ上に重ねて表示できる | 必須 |
| GRP-2 | シグナルごとにサブプロット（縦並び）で表示できる | 必須 |
| GRP-3 | 時間軸のズーム（拡大/縮小）をマウスホイール・ドラッグで行える | 必須 |
| GRP-4 | カーソルを合わせてタイムスタンプと値をツールチップで読み取れる | 必須 |
| GRP-5 | 時間範囲をドラッグ選択できる | 必須 |
| GRP-6 | 選択した時間範囲のデータを ASC 形式でエクスポートできる | 推奨 |
| GRP-7 | グラフを PNG 画像として保存できる | 必須 |
| GRP-8 | グラフを SVG 画像として保存できる | 推奨 |
| GRP-9 | Y 軸のスケールを自動調整 / 手動設定できる | 推奨 |
| GRP-10 | 物理値（DBC デコード後）と Raw 値の切替表示ができる | 推奨 |
| GRP-11 | 凡例を表示し、クリックでシグナルの表示/非表示を切替できる | 必須 |
| GRP-12 | フレーム送信途絶を可視化する: 隣接サンプル間隔が DBC/ARXML 定義の送信周期の 5 倍を超える区間はライン描画を行わない | 必須 |
| GRP-13 | 凡例（または凡例 UI）でシグナルをクリック選択して強調表示できる。複数選択可、非選択シグナルは不透過率 30% で描画、空クリックでリセット。静的 SVG 制約のため Plotly 外の Flet UI として実装する | 必須 |

### 3.6 フレーム抽出エクスポート

指定条件に合致するフレームのみを抽出し、ASC ファイルとして保存する。

| 要件 | 説明 | 優先度 |
|------|------|--------|
| EXP-1 | フレーム ID（複数指定可）でフィルタしてエクスポートできる | 必須 |
| EXP-2 | フレーム名（DBC 解決後）でフィルタしてエクスポートできる | 必須 |
| EXP-3 | チャンネル（バス番号）でフィルタしてエクスポートできる | 必須 |
| EXP-4 | 時間範囲を指定してエクスポートできる | 必須 |
| EXP-5 | エクスポート先ファイル名を指定できる | 必須 |
| EXP-6 | 元の ASC ファイルのヘッダ情報を保持してエクスポートする | 必須 |
| EXP-7 | エクスポートの進捗をプログレスバーで表示する | 推奨 |
| EXP-8 | 複数フィルタ条件を AND で組み合わせてエクスポートできる | 必須 |

### 3.7 統計情報

フレーム単位の統計情報を算出・表示する。

| 要件 | 説明 | 優先度 |
|------|------|--------|
| STA-1 | フレームごとのメッセージ受信回数を表示する | 必須 |
| STA-2 | フレームごとの送信周期（平均・最小・最大・標準偏差）を算出する | 必須 |
| STA-3 | 周期ジッタ（送信周期のばらつき）を表示する | 推奨 |
| STA-4 | チャンネルごとのバスロード（%）を時間帯別に算出する | 必須 |
| STA-5 | バスロードの時系列グラフを表示できる | 推奨 |
| STA-6 | 統計情報を CSV でエクスポートできる | 推奨 |
| STA-7 | 時間範囲を指定して統計を算出できる | 推奨 |

### 3.8 リアルタイム CAN 受信

Vector VN1610 インターフェースを使用して CAN FD バスをリアルタイムに受信する。

| RT-2 | ボーレート（Arbitration / Data）を設定できる | 必須 |
| RT-3 | 受信フレームをリアルタイムにトレースビューに表示する | 必須 |

## 5. データモデル

### 5.1 CAN フレーム

```python
@dataclass
class CanFrame:
    """パース済み CAN/CAN FD フレーム"""
    timestamp: float          # 秒（絶対時刻）
    channel: int              # チャンネル番号 (1, 2, 3, ...)
    arbitration_id: int       # フレーム ID
    is_extended_id: bool      # 拡張 ID (29bit) フラグ
    is_fd: bool               # CAN FD フラグ
    is_rx: bool               # Rx=True, Tx=False
    dlc: int                  # DLC (0-15)
    data_length: int          # 実データバイト数 (0-64)
    data: bytes               # データバイト列
    frame_name: Optional[str] # DBC 解決後のフレーム名
    # CAN FD 固有フィールド
    brs: Optional[bool]       # Bit Rate Switch
    esi: Optional[bool]       # Error State Indicator
```

### 5.2 デコード済みシグナル値

```python
@dataclass
class SignalValue:
    """DBC デコード後のシグナル値"""
    signal_name: str          # シグナル名
    raw_value: int            # Raw 値
    physical_value: float     # 物理値 (factor * raw + offset)
    unit: str                 # 単位
    timestamp: float          # タイムスタンプ
    frame_id: int             # 元フレーム ID
```

### 5.3 統計データ

```python
@dataclass
class FrameStatistics:
    """フレーム単位の統計情報"""
    arbitration_id: int
    frame_name: Optional[str]
    channel: int
    count: int                # 受信回数
    cycle_avg_ms: float       # 平均周期 (ms)
    cycle_min_ms: float       # 最小周期 (ms)
    cycle_max_ms: float       # 最大周期 (ms)
    cycle_std_ms: float       # 周期標準偏差 (ms)
    first_timestamp: float    # 最初の受信時刻
    last_timestamp: float     # 最後の受信時刻
```

### 6.2 ASC フォーマット仕様

Vector ASC 形式のパース仕様。CANoe 17.x が出力するフォーマットに準拠する。

#### 6.2.1 ヘッダ

```
date {曜日} {月} {日} {HH:MM:SS.mmm} {am/pm} {年}
base {hex|dec}
timestamps {absolute|relative}
internal events logged
// version {version_string}
// Measurement UUID: {uuid}
Begin TriggerBlock {曜日} {月} {日} {HH:MM:SS.mmm} {am/pm} {年}
```

#### 6.2.2 Classic CAN フレーム

```
{timestamp} {channel}  {id}        {Rx|Tx}   d {dlc} {data_bytes}  Length = {length} BitCount = {bitcount} ID = {decimal_id}
```

**フィールド定義:**

| フィールド | 型 | 説明 | 例 |
|-----------|-----|------|-----|
| timestamp | float | 秒（小数点以下 6 桁） | `0.005454` |
| channel | int | チャンネル番号 | `1` |
| id | hex string | フレーム ID（拡張 ID は末尾 `x`） | `AFACE31x` |
| Rx/Tx | string | 送受信方向 | `Rx` |
| d | char | データフレーム識別子 | `d` |
| dlc | int | データ長 | `8` |
| data_bytes | hex | スペース区切りデータ | `4C B0 00 00 00 00 00 00` |
| Length | int | ビット長（informational） | `1103547` |
| BitCount | int | ビットカウント | `142` |
| ID | int + x | 10進 ID（拡張は末尾 `x`） | `184208945x` |

**サンプル:**
```
   0.005454 1  AFACE31x        Rx   d 8 4C B0 00 00 00 00 00 00  Length = 1103547 BitCount = 142 ID = 184208945x
```

#### 6.2.3 CAN FD フレーム

```
{timestamp} CANFD   {channel} {Rx|Tx} {id} {frame_name} {brs} {esi} {dlc_code} {data_length} {data_bytes}   {bit_timing_fields}
```

**フィールド定義:**

| フィールド | 型 | 説明 | 例 |
|-----------|-----|------|-----|
| timestamp | float | 秒（小数点以下 6 桁） | `0.004603` |
| `CANFD` | literal | CAN FD フレーム識別子 | `CANFD` |
| channel | int | チャンネル番号 | `2` |
| Rx/Tx | string | 送受信方向 | `Rx` |
| id | hex string | フレーム ID（拡張 ID は末尾 `x`） | `29c`, `cd9ad5cx` |
| frame_name | string | フレーム名（DBC 未定義時は空白列） | `VSA_29C` |
| brs | int | Bit Rate Switch (0/1) | `1` |
| esi | int | Error State Indicator (0/1) | `0` |
| dlc_code | hex | DLC コード (0-f) | `8`, `f` |
| data_length | int | 実データバイト数 | `8`, `64` |
| data_bytes | hex | スペース区切りデータ | `00 07 dc a9 48 aa 6c 20` |
| bit_timing | int... | ビットタイミング情報（複数フィールド） | `100704  130   303000 b000809a ...` |

**DLC コード → データ長マッピング（CAN FD）:**

| DLC コード | データ長 (bytes) |
|-----------|-----------------|
| 0–8 | 0–8 |
| 9 | 12 |
| a | 16 |
| b | 20 |
| c | 24 |
| d | 32 |
| e | 48 |
| f | 64 |

**サンプル（8 バイト）:**
```
   0.004603 CANFD   2 Rx        29c  VSA_29C                          1 0 8  8 00 07 dc a9 48 aa 6c 20   100704  130   303000 b000809a 46500250 4b280150 20011736 2000091c
```

**サンプル（64 バイト）:**
```
   0.018189 CANFD   2 Rx    cd9ad5cx BDCAN1_0CD9AD5C                  1 0 f 64 00 00 40 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 21 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00   413203  698   303000 b0117866 46500250 4b280150 20011736 2000091c
```

#### 6.2.4 特殊行

| パターン | 説明 |
|----------|------|
| `Start of measurement` | 計測開始マーカー |
| `End TriggerBlock` | トリガーブロック終了 |
| `//` で始まる行 | コメント（version, UUID 等） |

### 6.3 大容量ファイル対応設計

GB 級のログファイルを扱うための設計。

#### 6.3.1 ストリーミングパーサ

- ファイルをメモリに全展開しない
- 行単位でストリーミングパースし、フレームデータを生成
- 初回読込時にインデックスファイル（`.asc.idx`）を生成

#### 6.3.2 インデックスファイル

```python
@dataclass
class AscIndex:
    """ASC ファイルインデックス"""
    file_path: str
    file_size: int
    file_hash: str               # ファイル変更検出用ハッシュ
    total_frames: int
    time_range: Tuple[float, float]  # (start, end) 秒
    channels: List[int]
    frame_id_offsets: Dict[int, List[int]]  # frame_id → [file_offset, ...]
    chunk_offsets: List[Tuple[int, float]]  # [(file_offset, timestamp), ...] 1000行ごと
```

- 初回読込時に自動生成（`.asc.idx` として保存）
- 次回読込時はインデックスの整合性を確認し、再利用
- フレーム ID 指定アクセス・時間範囲指定アクセスを O(1)〜O(log n) で実現

#### 6.3.3 表示用データの遅延読込

- トレースビュー: 表示中の行範囲のみをファイルから読み込む（仮想スクロール）
- グラフ: 指定シグナルに関連するフレームのみをインデックスから抽出して読み込む
- 統計: 全フレームをストリーミングスキャンして算出（メモリには統計値のみ保持）

### 6.4 スレッドモデル

```
メインスレッド (Flet UI)
  │
  ├── ワーカースレッド: ASC パース / インデックス生成
  │     └── Queue → UI 更新 (page.update)
  │
  ├── ワーカースレッド: 統計計算
  │     └── Queue → UI 更新
  │
  └── ワーカースレッド: リアルタイム CAN 受信
        └── Queue → トレース追記 / グラフ更新
```

- UI 操作はメインスレッドで処理
- ファイル I/O・計算処理はワーカースレッドで非同期実行
- `threading.Thread` + `queue.Queue` でスレッド間通信
- リアルタイム受信は専用スレッドで常時ループ

### 6.5 リアルタイム CAN 受信

#### 6.5.1 python-can 設定

```python
import can

bus = can.Bus(
    interface='vector',
    channel=0,                    # VN1610 チャンネル
    bitrate=500000,               # Arbitration 500 kbps
    fd=True,
    data_bitrate=2000000,         # Data 2 Mbps
    app_name='CAN_FD_Analyzer',
)
```

#### 6.5.2 受信ループ

```python
def receive_loop(bus: can.Bus, frame_queue: Queue):
    """ワーカースレッドで実行する受信ループ"""
    while not stop_event.is_set():
        msg = bus.recv(timeout=0.1)
        if msg is not None:
            frame = convert_to_can_frame(msg)
            frame_queue.put(frame)
```

#### 6.5.3 Vector XL Driver 要件

- Vector XL Driver Library がインストールされていること
- VN1610（または互換デバイス）が USB 接続されていること
- python-can の Vector インターフェースが利用可能であること
- Vector Hardware Config で該当チャンネルがアプリケーションに割り当てられていること

### 6.6 パフォーマンス目標

| 項目 | 目標値 |

| ASC 読込速度（インデックス生成） | ≥ 50 MB/s |

## 8. 操作仕様

### 8.1 基本操作フロー

```
1. [ASC 開く] → ファイル選択 → パース開始（プログレス表示）
2. [DBC 開く] → DBC ファイル選択（複数可） → フレーム名/シグナル解決
3. トレースタブでフレーム一覧を確認・フィルタリング
4. シグナルツリーで対象シグナルを選択
5. グラフタブでシグナル値の時系列変化を確認
6. 必要に応じてフレーム抽出エクスポート
```

### 8.2 キーボードショートカット

| ショートカット | 機能 |
|---------------|------|
| Ctrl+O | ASC ファイルを開く |
| Ctrl+D | DBC ファイルを開く |
| Ctrl+E | フレーム抽出エクスポート |
| Ctrl+S | グラフ画像保存 |
| Ctrl+F | フィルタ/検索バーにフォーカス |
| Ctrl+1 | トレースタブに切替 |
| Ctrl+2 | グラフタブに切替 |
| Ctrl+3 | 統計タブに切替 |
| Home | トレースビュー先頭へ |
| End | トレースビュー末尾へ |

### 8.3 エラーハンドリング

| エラー | 対処 |
|--------|------|
| ASC ファイルフォーマット不正 | エラー行をスキップし、スナックバーで通知（解析は継続） |
| DBC ファイル読込失敗 | エラーダイアログ表示、ID + Raw データのみで動作継続 |
| Vector デバイス未接続 | 接続ダイアログでエラー表示、オフラインモードで動作 |
| メモリ不足 | インデックスモードへ自動切替、ユーザーに通知 |
| パース不能行（未知フォーマット） | 行をスキップし、ログに警告出力 |

## 8.4 設定ファイル (Config)

### 8.4.1 概要

ユーザ設定（選択シグナル等）を JSON ファイル（拡張子 `.canalzcfg`）として保存・読込する。

| 要件 | 説明 | 優先度 |
|------|------|--------|
| CFG-1 | 選択中シグナルリスト (frame_id, signal_name) を保存できる | 必須 |
| CFG-2 | 保存した設定ファイルを読み込んで選択状態を復元できる | 必須 |
| CFG-3 | 将来拡張（フィルタ条件・レイアウト等）に備え、JSON の構造は version フィールドを持つ | 必須 |

### 8.4.2 ファイル構造（v1）

```json
{
  "version": 1,
  "selected_signals": [
    {"frame_id": 344, "signal_name": "EAT_TRANS_SPEED"},
    ...
  ]
}
```

### 8.4.3 CANoe Config (.cfg) との共用

**共用不可**。CANoe の `.cfg` は Vector 独自のバイナリ形式で公開仕様がないため、直接読み書きは行わない。DBC / ARXML ファイルの参照は共通化できるが、パネル選択状態等の UI 設定は本ツール独自 JSON とする。
